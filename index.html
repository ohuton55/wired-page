打ち込んだ数値の２倍と２乗を計算し、コメントする


                public class Math{
                  public static void main(String[] args){

                    //２倍にしてほしい数値を入力
                    twice(5);
                    twice(85);
                    twice(3247);
                    twice(57890101);

                    //２乗にしてほしい数値を入力
                    powerOfTow(10);
                  }

                  public static void twice(int n){
                    System.out.println(n + "の２倍は、");
                    System.out.println(n * 2);
                  }

                  public static void powerOfTow(int n){
                    int answer = 1;

                    for( int i=0 ; i< n ; i++){
                        answer = answer * 2;
                    }

                    System.out.println("２の" + n + "乗は、");
                    System.out.println(answer);
                  }

                }

今回はコメントも一応返してくれますが、文章自体にバリエーションがあるという訳ではなく　答えてくれる数値 を沢山出力している…といったもののようです。twice()　を増やした点以外はほぼお手本通りです。
powerOfTow();はもろにそうですね…なぜ増やさなかったかというと、大文字を打つことが少し苦手だから…です…。 たいへんよくない理由だと思うので、今後は大文字も含んだ名前をバンバンつけていこうと思います。

for文の新たな使い方として　　for(int i = j ; i < n ; i++){～}　という、条件内でさらに代入が出てくるという 初心者には頭の痛いプログラミングらしい文が出てきました。順をゆっくり追って考えていけば把握できるのですが…。 自分が組んでいないプログラムを読むことになった際に、１２０％ぐらい解釈を間違えてしまいそうで恐ろしいです。

それと　public static void main(String[] args){～}　のほかに、 また publicでstaticなvoidとして名前が付けられる 大きな箱（考え方がいまいちぼんやりなままですが…）を作れるという点も今回のポイントのようです。 どちらの引数(?)にもn　を使っていますが、ハコが違うので識別は全く問題無いみたいなんですね…。

answer = answer*2;　って　数学ではありえない文であり…　プログラムの文脈がわからないとただひたすら頭の中が？？？となるばかりで…難しいです！

打ち込んだ数値が３で割り切れるか判断し、コメントする


              public class ReturnTest{
                public static void main(String[] args){
                    System.out.println(isMultipleOf3(123456789));
                }
                public static String isMultipleOf3(int n){
                  String resultMessage = "";
                  if (n % 3 == 0){
                      resultMessage = n + "は３で割り切れます";
                  }else{
                      resultMessage = n + "は３で割り切れません";
                  }
                  return resultMessage;
                }

              }

今までで一番コード量が少ないので少し安心します。 メソッドの戻り値についての解説通りに、これまたお手本に沿って打ち込んだものです。直近のもの２つは 全く自分なりに打てていないので、反省しています。今後は復習の意味としても　とても大切だとは思うので、 できるだけ頑張るぞと思います。

isMultipleOf3()内に好きな数字を入れると瞬時に割り切れるかどうか判断してくれます。１０桁も余裕です。
そういえば１００桁だとどうなんでしょう…と思いついて、やってみたのですが、　「整数が大きすぎます」　 とのことでエラーになってしまいました。私のノート（Windows 64bit)ではどうも１０桁が限界のようです。スーパーコンピューターであれば 計算してしまうのでしょうか。　１０桁もの数を３で割るのが一瞬…という処理スピードは十分すごいと思いますが…
